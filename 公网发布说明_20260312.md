# Alpha AI 原生 App 公网发布说明

## 目标

把当前“局域网内测版”推进成：

- 手机随时随地可查看
- 支持其他用户下载安装
- 后端不再依赖本地电脑持续开机

## 已补齐的底座

- 后端已支持环境变量控制 `APP_CORS_ORIGINS`
- 已补 `Dockerfile` 和 `requirements.server.txt`
- 已补 `deploy_public.sh`
- 已补公网下载页 `public_release/index.html`
- 已补安卓发布脚本 `scripts/publish_android_release.sh`
- 已补生产环境示例：
  - `.env.production.example`
  - `native_app/.env.production.example`

## 推荐上线结构

### 1. API 服务

- 域名: `https://api.example.com`
- 部署方式:
  - 单机 `systemd + nginx`
  - 或 Docker 容器

### 2. 下载页

- 域名: `https://download.example.com`
- 作用:
  - 给外部用户下载 APK
  - 展示版本号、发布时间、校验值

### 3. 原生包

- 构建时把 `EXPO_PUBLIC_API_BASE_URL` 改成公网 API
- 例如:

```bash
EXPO_PUBLIC_API_BASE_URL=https://api.example.com \
./gradlew assembleRelease
```

## 你后面怎么做

### 方案 A: 最快落地

1. 买一台云服务器
2. 准备两个域名:
   - `api.example.com`
   - `download.example.com`
3. 把 `.env.production.example` 复制成 `.env.production`
4. 填真实密码、密钥、CORS 域名
5. 运行:

```bash
./deploy_public.sh api.example.com download.example.com
```

6. 构建公网版 APK:

```bash
cd native_app
cp .env.production.example .env.production
```

把 `EXPO_PUBLIC_API_BASE_URL` 改成公网域名后再构建。

7. 发布安装包:

```bash
./scripts/publish_android_release.sh \
  native_app/android/app/build/outputs/apk/release/app-release.apk \
  1.0.1
```

然后把 `public_release/` 同步到下载站目录。

## 当前边界

- 代码和发布底座已补齐
- 还没有真正推到公网
- 因为当前没有服务器账号、域名、SSL 和外网访问条件
