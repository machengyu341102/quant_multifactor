# APP交互重构方案 2026-03-13

## 目标

把当前 App 从“功能集合”改成“交易决策产品”。

第一优先级不是加新功能，而是让用户在 5 秒内看懂 3 件事：

1. 现在最该看什么
2. 为什么看它
3. 下一步该怎么做

## 总体原则

### 1. 首页只做驾驶舱，不做信息堆场

- 首页第一屏只放最关键的 3 块：`最新推荐`、`风险提醒`、`持仓异动`
- 学习状态和微信镜像保留在第一屏下半区，但不再和推荐抢主视线
- 首页不负责解释一切，只负责分发动作

### 2. 推荐页负责决策，不负责讲故事

- 每条推荐都要给出：`结论`、`理由`、`风险`、`动作`
- 把“为什么推荐”前置，不要埋进详情页深处
- 主动作永远只有一个，其余动作作为次动作

### 3. 持仓页优先展示风险，不优先展示静态收益

- 排序规则从“代码顺序”改成“风险优先级”
- 接近止损、波动异常、数据过期的仓位永远排前
- 页面第一眼应该让用户知道“哪个仓位先处理”

### 4. 决策台是真正的脑子，不是系统监控台

- 第一块是交互诊股
- 第二块是系统建议
- 第三块是今日学习进度
- 第四块才是深层指标和策略热区

### 5. 所有页面统一一套动作语言

- 蓝色：主动作
- 绿色：完成或正向状态
- 橙色：观察、待处理
- 红色：风险、止损、异常

## 导航结构

底部导航保留 5 个主入口，不再扩充：

1. 首页
2. 推荐
3. 持仓
4. 决策台
5. 我的

辅助页面放二级层：

- 提醒中心
- 微信镜像消息中心
- 交易回执
- 反馈窗
- 运维诊断

## 新版页面结构

## 首页

目标：一眼看到“现在最重要的事”。

### 第一屏结构

1. 顶部状态条
- 系统运行状态
- 数据新鲜度
- 今日是否有新推荐

2. 今日主任务
- 只显示 1 条最值得处理的事项
- 例：`000001 接近止损，优先处理`
- 主按钮：`立即处理`

3. 最新推荐
- 只显示 1 条首推推荐
- 字段：代码、名称、评分、时间、止损、目标
- 主按钮：`看详情`
- 次按钮：`去诊股`

4. 持仓异动
- 只显示最危险的 1 到 2 个持仓
- 字段：当前价、止损价、偏离幅度、建议动作
- 主按钮：`调整风险`

### 第一屏下半区

5. 今日学习
- 今日学习轮次
- 因子调整
- 在线更新
- 日日精进是否完成
- 主按钮：`去决策台`

6. 微信镜像
- 最近 1 条推送
- 标注类型：推荐 / 告警 / 报告
- 主按钮：`打开消息中心`

### 首页不应该再出现的东西

- 一大排平级统计卡抢视线
- 过多策略列表
- 大段说明文案
- “系统总览”这类后台式标题

## 推荐页

目标：把推荐变成可判断、可执行的决策。

### 列表卡结构

每条推荐卡固定这 4 层：

1. 结论
- `可开仓`
- `继续观察`
- `不建议追`

2. 理由
- 只保留 2 到 3 个高信号理由
- 例：趋势转强、资金净流入、位置仍安全

3. 风险
- 止损位
- 失效条件
- 数据时间

4. 动作
- 主动作：`开仓`
- 次动作：`去诊股`
- 次动作：`加入观察`

### 详情页结构

1. 结论区
2. 评分拆解区
3. 风险区
4. 执行动作区
5. 反馈入口

### 推荐页新增规则

- 如果当天没有新推荐，顶部明确显示：`今天无新推荐，下面展示最近一次有效推荐`
- 每条推荐都显示生成时间和适用窗口，避免把旧信号误当新信号

## 持仓页

目标：优先处理风险，不优先看数字好不好看。

### 列表排序

排序从高到低：

1. 接近止损
2. 当日大幅波动
3. 盈利回撤明显
4. 数据过期
5. 普通持仓

### 卡片结构

1. 代码与名称
2. 当前盈亏
3. 风险状态
4. 当前价 / 止损 / 止盈
5. 建议动作

### 详情页结构

1. 仓位摘要
- 成本
- 数量
- 当前价
- 浮盈亏

2. 风险控制
- 当前止损
- 当前止盈
- 距离止损百分比

3. 系统建议
- 继续持有
- 上移止损
- 部分减仓
- 全部平仓

4. 固定底部动作条
- `调整风险`
- `减仓`
- `平仓`

## 决策台

目标：真正像“脑子”，而不是一页后台统计。

### 第一屏结构

1. 交互诊股
- 输入股票代码
- 快速候选
- 开始诊股

2. 诊股结论
- 多头 / 观察 / 回避
- 置信度
- 当前价格
- 主按钮：`加入推荐观察`

3. 系统建议
- 今天最该补的系统动作
- 例：`学习链未完成，建议先跑日日精进`

4. 今日学习进度
- 今日学习轮次
- 因子调整
- 在线更新
- 上次执行时间
- 主按钮：`立即执行`

### 决策台不该做什么

- 不要先堆策略热区
- 不要先堆大量技术指标
- 不要让用户先读很多数字再看到结论

## 我的

目标：收口账户、配置和后台能力，不干扰主交易链。

### 保留内容

- 账号信息
- 通知权限
- API 地址
- 反馈入口
- 消息中心入口
- 运维诊断入口

### 下沉内容

以下内容不要再放首页：

- 运行配置
- 调试项
- 权限设置
- 推送测试

## 状态表达系统

所有关键区块都必须显示时间，不允许“只有数字没有时间”。

### 必须统一显示的状态

- 最后更新时间
- 数据是否新鲜
- 是否有新信号
- 是否已完成学习
- 是否有待处理风险

### 数据新鲜度规则

- 5 分钟内：正常
- 5 到 15 分钟：提醒
- 超过 15 分钟：标记可能过期

### 学习链规则

- 今日未完成：橙色
- 已完成：绿色
- 执行失败：红色

## 交互组件系统

统一收口成 6 类核心组件：

1. 主任务卡
2. 推荐结论卡
3. 风险卡
4. 学习进度卡
5. 微信镜像卡
6. 固定底部动作条

这样做的目的不是好看，而是让用户形成稳定操作习惯。

## 文案系统

少用后台词，多用动作词。

### 统一替换

- `系统总览` -> `现在先看这几件事`
- `学习状态` -> `今天脑子更新到哪了`
- `信号` -> `推荐`
- `系统建议` -> `现在建议这么做`
- `健康摘要` -> `系统现在稳不稳`

## 实施顺序

为了保证整系统稳定，改造顺序这样排：

1. 首页重构
- 文件：
  - `native_app/app/(tabs)/index.tsx`

2. 推荐页重构
- 文件：
  - `native_app/app/(tabs)/signals.tsx`
  - `native_app/app/signal/[id].tsx`

3. 持仓页重构
- 文件：
  - `native_app/app/(tabs)/positions.tsx`
  - `native_app/app/position/[code].tsx`

4. 决策台重构
- 文件：
  - `native_app/app/(tabs)/brain.tsx`

5. 我的页收口
- 文件：
  - `native_app/app/(tabs)/profile.tsx`
  - `native_app/app/messages.tsx`
  - `native_app/app/ops.tsx`

## 不先动的链路

为了保证登录、交易动作、鉴权稳定，这几块先不碰：

- 登录逻辑
- token 存储
- 开仓接口
- 调整止损接口
- 平仓接口
- 推送 token 注册逻辑

## 交付标准

这一版改完后，至少要达到：

1. 首页第一屏就能看到推荐、风险、学习、微信镜像
2. 推荐页每条卡都能直接给结论和动作
3. 持仓页第一眼能知道哪个仓位最危险
4. 决策台第一屏就能诊股和触发学习
5. 所有关键模块都有时间和状态标记

## 一句话结论

这次不是“把页面做漂亮”，而是把 App 重排成一套稳定的决策链：

`首页看优先级 -> 推荐做判断 -> 持仓做处理 -> 决策台补脑子 -> 我的页做配置`
