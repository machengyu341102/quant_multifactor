# Alpha AI 交易系统全貌单页

更新时间：`2026-03-13`

## 1. 系统定位

`Alpha AI` 不是单一选股器，也不是普通行情 App。  
它当前的定位是：

**面向内部投研与交易执行的 AI 决策操作系统原型**

核心目标有三件事：

- 把市场信号、事件驱动、强势股和风控建议放进同一决策界面
- 把推荐、诊股、持仓、反馈、学习做成闭环
- 让系统具备“持续学习、持续修正、持续进化”的能力

## 2. 当前系统结构

```text
市场/行情/事件数据
        ↓
策略与信号层
  - 隔夜选股
  - 集合竞价
  - 趋势跟踪
  - 低吸回调
  - 事件/板块异动
        ↓
决策增强层
  - 个股诊断
  - 强势收益引擎
  - 主线资金迁移 / 主题雷达
  - 风险提醒
        ↓
执行与反馈层
  - 模拟开仓 / 调止损 / 减仓 / 平仓
  - 动作回执
  - 交易记录
  - 用户反馈
        ↓
学习与优化层
  - 信号验证
  - 因子调优
  - 在线更新
  - 日日精进
  - ML 因子模型
        ↓
原生 App 展示层
  - 首页指挥台
  - 推荐页
  - 持仓页
  - 决策台
  - 控制中心
```

## 4. App 当前能演示什么

当前原生 App 已经可以完整演示以下链路：

1. 首页看今日焦点、推荐、风险和学习状态
2. 推荐页看结论、理由、风险和动作
3. 决策台做现场诊股，看主题资金迁移与系统建议
4. 持仓页做调止损、减仓、平仓
5. 消息中心看微信镜像、风控和周报
6. 反馈页收集用户意见，但最终是否采纳由决策账号决定

## 5. 这套系统的差异化

### 不是只看“今天谁涨”

系统已经开始从以下维度合成决策：

- 策略信号
- 事件与板块异动
- 强势股持续性
- 风险与止损结构
- 学习反馈

### 不是只会推荐，还能闭环

它不是“发一条信号就结束”，而是：

`推荐 -> 诊股 -> 执行 -> 回执 -> 记录 -> 学习 -> 再修正`

### 不是平均用力，而是有主次

当前学习资源已经明显向核心主策略集中：

- 第一梯队：`隔夜 / 集合竞价 / 趋势跟踪`
- 第二梯队：`低吸 / 缩量 / 尾盘`
- 弱策略进入冻结或监控态

## 6. 当前边界

当前阶段更准确的表述是：

**可运行、可演示、可持续迭代的交易决策系统原型**

而不是：

- 机构级生产交易平台
- 对外公开商业化投顾产品
- 99.99% SLA 产线系统

当前明确边界：

- 仍以本地和内测形态为主
- 组合当前为空仓，更适合演示推荐、主线、诊股、学习，不适合演示当前持仓收益
- 合规层面仍应按“内部投研/辅助决策工具”口径表达

## 7. 下阶段最值钱的方向

下一阶段最值钱，不是继续堆页面，而是把三件事再做深：

### 1. 事件总控

把战争、原油、制裁、政策、汇率等因素从“边缘策略”升级成“系统上层风险与主线总控”。

### 2. 强势股中台

把连板、龙头、主升浪、波段延续统一成一套“持续性评分与持有管理”引擎。

### 3. 仓位与防守引擎

把总仓位、单票上限、分仓、分批进出、回撤控制真正落进系统，不只停留在人工经验。

## 8. 现在最合适的对外表述

对基金公司或机构交流时，当前最适合的表达是：

**一个正在成型的 AI 交易决策操作系统，已经具备推荐、诊股、主线发现、风控提醒、学习闭环和原生 App 承载能力。**

如果要更克制一点，可以直接说：

**这是一个面向内部投研与交易辅助的智能交易系统原型，重点不在单一荐股，而在“持续学习的决策闭环”。**
