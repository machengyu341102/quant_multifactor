# 产业资本决策操作系统架构与路线图

## 一句话定位

把系统从“交易软件”升级成“方向研判 + 产业跟踪 + 资本配置 + 事业布局”的统一中枢。

## 目标形态

系统最终输出不应只有股票推荐，还应同时输出四类结果：

1. 交易主线：今天和这周该打哪条线，适合主线观察、中期波段还是连涨接力。
2. 产业方向：未来 1-3 个月、1-3 年值得持续跟踪的方向。
3. 事业建议：该进入哪个环节做调研、合作、供应链布局或轻资产卡位。
4. 风险约束：哪些方向暂时只能观察，哪些方向存在政策兑现、供需逆转或地缘扰动风险。

## 核心架构

### 1. 国家方向引擎

- 输入：两会、五年规划、部委政策、产业政策、地缘政治、制裁与反制。
- 输出：政策方向、政策桶、事件偏向、方向强度。

### 2. 产业链演化引擎

- 输入：供需、涨价降价、产能、替代、上下游传导、产业阶段。
- 输出：行业阶段、需求侧驱动、供给侧驱动、兑现链、产业链位置。

### 3. 主线与资金引擎

- 输入：板块异动、资金偏好、强势股持续性、市场关注度、主题扩散。
- 输出：主线种子、中期波段、连涨接力、主线阶段。

### 4. 资本执行引擎

- 输入：综合推荐、事件总控、仓位纪律、主题上限、单票上限。
- 输出：首仓、总仓、单主题上限、是否允许追击或减仓。

### 5. 事业布局引擎

- 输入：国家方向、产业链阶段、兑现链、资金偏好。
- 输出：调研清单、合作环节、重点跟踪名单、先观察/先卡位/中线推进建议。

## 当前已落地

- 政策方向雷达
- 主线发现与阶段引擎
- 综合推荐榜
- 仓位与分仓防守引擎
- 产业资本方向层

当前系统已经能把：

`政策/地缘政治 -> 产业链 -> 主线 -> 资金偏好 -> 个股候选 -> 仓位建议`

压成一条连续链路。

## 下一阶段优先级

### P1：官方政策知识库

- 只吃官方源：政府工作报告、五年规划、发改委、工信部、财政部、国资委、国家数据局等。
- 目标：把“事件代理”升级成“政策原文与方向映射”。

### P2：行业阶段跟踪

- 对重点方向持续输出：早期、升温、扩散、兑现、承压。
- 目标：解决“知道方向，但不知道阶段”的问题。

### P3：产业兑现链

- 从“提法”跟到“预算、项目、订单、价格、产能、利润、业绩”。
- 目标：判断一条线是概念炒作还是开始兑现。

### P4：事业动作清单

- 每条方向给出“能做什么”，而不只是“能买什么”。
- 目标：让系统同时服务交易和事业发展。

## 先聚焦的方向

建议先深做 5 条线：

1. AI 与数字基础设施
2. 国产替代与自主可控
3. 中美博弈与反制链
4. 能源安全与资源涨价链
5. 高端制造与军工安全

## 核心原则

- 看大做小：先判断方向，再找个股和环节。
- 先主线后买点：先决定该不该做，再决定怎么买。
- 金融验证认知：用市场资金和股价验证方向判断。
- 实业兑现价值：用订单、价格、项目和利润验证长期价值。

## 预期结果

如果这条路做深，系统就不只是“交易软件”，而会变成：

`产业资本决策操作系统`

它既能指导交易，也能指导事业方向与长期布局。
