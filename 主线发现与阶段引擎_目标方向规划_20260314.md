# 主线发现与阶段引擎 目标方向规划

## 核心原则

看大做小，先定方向，再找机会。

系统以后不是先问“哪只票强”，而是先问：

- 哪条主线最值得先看
- 这条主线现在处在哪个阶段
- 该用主线观察、中期波段还是连涨接力去做
- 哪些后排票该直接回避

## 目标方向

### 1. 大方向总控

用国家战略、政策方向、国际事件、商品价格、产业趋势，先确定今天的大方向。

输出：

- 今日最强方向
- 回避方向
- 方向偏多 / 偏空 / 中性
- 总仓和主题上限建议

### 2. 主线发现与阶段引擎

把一条主线拆成 5 个支撑分：

- 政策/事件代理分
- 趋势分
- 关注度分
- 资金偏好分
- 阶段分

输出：

- 方向分
- 阶段标签：早期孵化 / 中期扩散 / 主升波段 / 高位分歧
- 参与方式：主线观察 / 中期波段 / 连涨接力 / 后排回避

### 3. 个股机会层

在主线内部再做个股筛选，不是平均撒网。

分成三层：

- 主线种子
- 中期波段 / 连涨候选
- 策略候选

输出：

- 焦点票
- 原因
- 建议首仓
- 什么时候该观察，什么时候该动手

### 4. 仓位执行层

方向对了，还要把仓位做对。

输出：

- 总仓建议
- 单票上限
- 单主题上限
- 首仓比例
- 减仓 / 锁盈 / 回避加仓

### 5. 学习复盘层

复盘不只看“涨没涨”，而要看：

- 这条主线在哪个阶段最容易赚钱
- 哪些方向只是热度，不是趋势
- 哪些票适合波段，哪些只适合观察

输出：

- 主线阶段回放
- 参与方式胜率
- 方向判断偏差
- 权重更新

## 近期落地顺序

### 第一阶段

先把当前已经做出来的影子层跑稳：

- 主线发现与阶段引擎
- 主线种子 / 中期波段 / 策略候选分层
- 事件总控进入综合排序和仓位建议

### 第二阶段

把政策方向做硬，不只吃新闻关键词：

- 政府工作报告
- 十五五规划
- 两会
- 部委产业政策
- 国家战略

### 第三阶段

把行业发展周期和阶段判断做成标准化：

- 导入期
- 渗透期
- 加速期
- 成熟期
- 退潮期

### 第四阶段

把方向层正式接到执行层：

- 哪些方向今天允许打
- 哪些方向只能观察
- 哪些方向该直接回避

## 一句话目标

把系统从“会推荐股票”，推进成“会判断方向、识别阶段、聚焦主线、控制仓位”的交易决策操作系统。
